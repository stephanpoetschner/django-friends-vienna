# docker-py ![whale](img/docker.png) Dra탑en Lu훾anin --- ## What's [docker](http://www.docker.io)? > an open source project to pack, ship and run any application > as a lightweight container - manager for (Linux) containers - like virtualisation, but: - no hardware emulation - fast --- # BAM! ```shell $ docker run -i -t ubuntu bash root@6b5542600661:/# ``` --- ## Images - [image registry](https://index.docker.io/) ```shell $ docker commit myname/kickassapp $ docker push myname/kickassapp ``` - Dockerfile - build it yourself ``` FROM ubuntu RUN apt-get install -y python-lxml python-imaging python-jinja2 git make RUN git clone https://github.com/pagekite/Mailpile.git ADD start.sh /start.sh CMD ./start.sh EXPOSE 33411 ``` --- ## Why should I use it? - true sandboxing (>virtualenv) - crazy deployment possibilities (think Lego) - create your own Heroku - [dokku](https://github.com/progrium/dokku), [flynn](https://flynn.io/) --- ## [docker-py](https://github.com/dotcloud/docker-py) - docker daemon offers an API - docker-py = Python API wrapper - fairly well-maintained - why should I use it? --- # #automate all the things --- ## My use-case: [Cloudfleet](https://cloudfleet.io/) ![cloudfleet](img/cloudfleet.png) - home server for easy hosting of essential internet services - docker containers - on-premise software demo --- ## docker-py example import docker c = docker.Client(base_url='http://localhost:4444', version='1.6', timeout=10) container = c.create_container('kermit/hellonode', name='mycontainer', ports=[1337]) c.start(container, publish_all_ports=True) --- ## Impressions - bugs - [stop hangs](https://github.com/dotcloud/docker/issues/1300) - [ports not exposed](https://github.com/dotcloud/docker-py/issues/141) - no remote API auth (yet) - SSH tunneling - self-signed HTTPS certificates - great when it works - very promising! - docker 1.0 coming soon - a very active open source project --- ## Thanks!
